## When to Use Each Package

### Start with @magicyan/core
If you need general-purpose utilities that work in any TypeScript project:
- String manipulation and formatting
- Data validation
- Date/time utilities
- Math helpers

### Use @magicyan/discord for Discord Bots
When building Discord bots and need:
- Channel and member utilities
- Command helpers
- Embed builders
- Component creators
- Role management

### Add @magicyan/discord-events for Advanced Events
When you need events beyond standard Discord.js:
- Member timeout events
- Voice state changes (connect/disconnect/move)
- Extended role events
- Webhook message detection
- User kick/ban events

### Use @magicyan/discord-ui for Interactive Components
When building interactive bot features:
- Paginated embeds
- Multi-page menus
- Confirmation dialogs
- Custom button layouts

### Use @magicyan/canvas for Image Generation
When creating custom images:
- Welcome cards
- User profiles
- Server banners
- Custom graphics with gradients and filters

### Use @magicyan/minecraft for Minecraft Integration
When integrating Minecraft features:
- Player profile lookups
- Server status checks
- Skin rendering
- Mojang API integration

## Installation

All packages are published on npm under the `@magicyan` scope. Install only what you need:

<CodeGroup>
```bash npm
npm install @magicyan/core
npm install @magicyan/discord
npm install @magicyan/discord-events
npm install @magicyan/discord-ui
npm install @magicyan/canvas
npm install @magicyan/minecraft
```

```bash pnpm
pnpm add @magicyan/core
pnpm add @magicyan/discord
pnpm add @magicyan/discord-events
pnpm add @magicyan/discord-ui
pnpm add @magicyan/canvas
pnpm add @magicyan/minecraft
```

```bash yarn
yarn add @magicyan/core
yarn add @magicyan/discord
yarn add @magicyan/discord-events
yarn add @magicyan/discord-ui
yarn add @magicyan/canvas
yarn add @magicyan/minecraft
```
</CodeGroup>

## TypeScript Support

All packages are written in TypeScript and include full type definitions. They support both ESM and CommonJS module formats.

<Note>
  Each package is independently versioned and can be used standalone or combined with other Magicyan packages.
</Note>
